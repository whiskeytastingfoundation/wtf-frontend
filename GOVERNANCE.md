# wtf-frontend Governance

This document describes how wtf-frontend is governed.

## Project Maintainers

The project is maintained by a group of individuals who have demonstrated a
commitment to the project through sustained contributions.

**Current Maintainers:**
- @mlieberman85

## Roles and Responsibilities

### Maintainers

Maintainers are responsible for:
- Reviewing and merging pull requests
- Triaging and responding to issues
- Making decisions about project direction
- Ensuring code quality and security
- Releasing new versions
- Enforcing the code of conduct

### Contributors

Anyone can contribute to wtf-frontend by:
- Submitting pull requests
- Reporting bugs
- Suggesting new features
- Improving documentation
- Helping other users in discussions

## Decision Making

Decisions are made by consensus among maintainers. For significant changes:
1. Open an issue to discuss the proposal
2. Allow time for feedback (typically 1 week for major changes)
3. Maintainers discuss and reach consensus
4. If no consensus, majority vote among maintainers decides

## Becoming a Maintainer

Contributors who have made significant, sustained contributions may be invited
to become maintainers. The process:

1. An existing maintainer nominates the contributor
2. Other maintainers discuss (privately or in an issue)
3. Decision by consensus or majority vote

We look for:
- Quality contributions over time
- Understanding of project goals and architecture
- Constructive participation in discussions
- Reliability and responsiveness

## Access and Security

- Maintainer access is reviewed periodically
- Inactive maintainers (6+ months) may have access removed
- All maintainers must have 2FA enabled on their GitHub accounts
- Sensitive operations (releases, security fixes) require review

## Code of Conduct

All participants must follow our [Code of Conduct](CODE_OF_CONDUCT.md).
Report violations to the maintainers.
